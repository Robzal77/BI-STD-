üîß Initial Setup (One-Time)

Step 1: Create ADO Wiki

Option A: Code Wiki (Recommended)

  1. Go to Azure DevOps ‚Üí Your Project ‚Üí Overview ‚Üí Wiki
  2. Click "Publish code as wiki"
  3. Select repository and folder (e.g., /Wiki)
  4. Name: "BI Factory Knowledge Base"
  5. Click "Publish"
üí° Code Wiki stores content in a Git repository, giving you version control!

Option B: Project Wiki

  1. Go to Azure DevOps ‚Üí Your Project ‚Üí Wiki
  2. Click "Create a wiki"
  3. Name: "BI Factory Docs"
  4. Click "Create"

Step 2: Get Wiki Repository URL

Navigate to your wiki and copy the clone URL:

# For Code Wiki: https://dev.azure.com/YourOrg/YourProject/_git/YourRepo # For Project Wiki: https://dev.azure.com/YourOrg/YourProject/_git/YourProject.wiki

Step 3: Clone Wiki Locally

# Navigate to BI-STD folder cd C:\Users\YourName\OneDrive\BI_Ops_Factory\BI-STD # Clone wiki repository git clone [Your-Wiki-URL] ./Wiki-ADO # Example: git clone https://dev.azure.com/ABInBev/BI-Project/_git/BI-Wiki ./Wiki-ADO
‚úì You should now have a Wiki-ADO/ folder in your BI-STD directory!

Step 4: Configure Git Credentials

Personal Access Token (PAT) - Recommended

  1. Go to ADO ‚Üí User Settings (gear icon) ‚Üí Personal Access Tokens
  2. Click "New Token"
  3. Configure:
    • Name: BI Wiki Sync
    • Scopes: Code (Read & Write)
    • Expiration: Custom (1 year)
  4. Click "Create" and copy the token
# Store credentials git config --global credential.helper store # First push will prompt: # Username: your.email@company.com # Password: [Paste PAT token here]
‚ö†Ô∏è Save your PAT! You won't be able to see it again after creation.

Step 3: Copy to ADO Wiki Clone

# Copy entire Wiki folder xcopy /E /I /Y Wiki\* Wiki-ADO\ # Or copy specific sections: xcopy /E /I /Y Wiki\Projects\* Wiki-ADO\Projects\ xcopy /E /I /Y Wiki\Standards\* Wiki-ADO\Standards\

Step 6: Verify in ADO

  1. Go to Azure DevOps ‚Üí Your Project ‚Üí Wiki
  2. Refresh the page
  3. Navigate to updated pages
  4. Verify formatting and links work correctly

‚öôÔ∏è Automation

Automated Sync Script

Create Automation/Sync_To_ADO_Wiki.bat to automate the process:

@echo off echo ======================================== echo Syncing Documentation to ADO Wiki echo ======================================== REM Step 1: Build local wiki echo. echo [1/4] Building local wiki content... cd "%~dp0" call "Sharable Documents.bat" REM Step 2: Copy to ADO wiki clone echo. echo [2/4] Copying to ADO wiki repository... xcopy /E /I /Y "..\Wiki\*" "..\Wiki-ADO\" REM Step 3: Git commit echo. echo [3/4] Committing changes... cd "..\Wiki-ADO" git add . git commit -m "Auto-sync: Updated BI documentation %date% %time%" REM Step 4: Push to ADO echo. echo [4/4] Pushing to Azure DevOps... git push origin main echo. echo ======================================== echo Wiki Sync Complete! echo View at: https://dev.azure.com/YourOrg/YourProject/_wiki echo ======================================== pause

Usage

  1. Save the script as Sync_To_ADO_Wiki.bat
  2. Update the URLs to match your ADO project
  3. Double-click to run whenever you want to publish
üí° Pro Tip: Add this to your weekly workflow - validate reports, then sync wiki!

üîß Troubleshooting

Issue Solution
Authentication Failed 1. Verify PAT is not expired
2. Check PAT has Code (Read & Write) permissions
3. Re-enter credentials: git credential reject
Push Rejected (Non-Fast-Forward) 1. Pull latest: git pull origin main
2. Resolve conflicts if any
3. Try push again: git push origin main
Wiki Pages Not Appearing 1. Check files are .md format
2. Verify .order file exists
3. Refresh ADO Wiki page (Ctrl+F5)
4. Check ADO Wiki permissions
Links Broken in Wiki 1. Use relative paths: [link](../folder/page)
2. Avoid spaces in filenames (use hyphens)
3. Check file extensions are correct
Images Not Displaying 1. Store images in wiki repository
2. Use relative paths to images
3. Check image file extensions
4. Verify images are committed and pushed

Quick Fixes

# Reset local wiki to match remote cd Wiki-ADO git fetch origin git reset --hard origin/main # Clear Git credentials git credential reject # Then re-enter on next push # Force push (use carefully!) git push --force origin main
‚ö†Ô∏è Force push with caution! It can overwrite others' changes. Coordinate with your team first.

üí° Best Practices

1. Keep Wiki Organized

  • Use consistent folder structure
  • Maintain .order files for navigation
  • Use descriptive file names (kebab-case)
  • Group related documentation together

2. Write Clear Commit Messages

# Good examples: git commit -m "Add Sales Dashboard documentation" git commit -m "Update governance standards - Added new DAX rules" git commit -m "Fix broken links in Finance report docs" # Bad examples: git commit -m "update" git commit -m "stuff" git commit -m "changes"

3. Review Before Pushing

  • Check markdown renders correctly locally
  • Test all internal links
  • Verify images display properly
  • Review git diff before committing

4. Use Branches for Major Updates

# Create feature branch git checkout -b wiki-update-2026-Q1 # Make changes and commit git add . git commit -m "Major wiki restructure for Q1" # Push branch git push origin wiki-update-2026-Q1 # Create Pull Request in ADO for team review

5. Regular Syncs

  • Sync after each governance check
  • Schedule weekly documentation updates
  • Keep local and remote in sync
